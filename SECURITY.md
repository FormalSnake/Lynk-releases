# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Lynk Browser, please report it responsibly by following these steps:

1. **Do not** create a public GitHub issue for security vulnerabilities
2. Send an email to the maintainers or reach out privately on [Discord](https://discord.gg/nwRrjPSaTM)
3. Include as much detail as possible:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (optional)

## What to Expect

- We will acknowledge receipt of your report within 48 hours
- We will provide updates on the status of your report
- We will notify you when the vulnerability has been fixed
- We will credit you in release notes (unless you prefer to remain anonymous)

## Scope

This security policy applies to the latest released version of Lynk Browser. Please ensure you are testing against the most recent release available on the [Releases](https://github.com/formalsnake/Lynk-releases/releases) page.

Thank you for helping keep Lynk Browser secure!
